测量性能工具

# 测量性能工具

建立具有低成本数据的高性能，弹性站点有几个核心目标。

对于每个目标，您都需要进行审核。

目的 为什么？ 要测试什么？
确保隐私，安全性和数据完整性，并支持强大的 API 使用 为什么 HTTPS 很重要 (opens new window) 为所有站点页面/路由和资产实施 HTTPS。
改善负载性能 53％的用户放弃 (opens new window)了加载时间超过三秒钟的网站 (opens new window) 可以异步加载或延迟加载的 JavaScript 和 CSS。为互动和有效负载大小设置时间目标：例如 3G 上的 TTI。设定绩效预算 (opens new window)。
减轻页面重量 •为具有上限数据计划的用户降低数据成本 •降低 Web 应用程序存储需求-对使用低规格设备的用户尤为重要 •降低托管和服务成本 •改善服务性能，可靠性和弹性 设置页面权重预算：例如，首次加载低于 400 kB。检查是否有大量 JavaScript。检查文件大小以查找肿的图像，媒体，HTML，CSS 和 JavaScript。查找可能会延迟加载的图像，并使用coverage 工具 (opens new window)检查未使用的代码 。
减少资源需求 •减少 延迟问题 (opens new window) •降低服务成本 •改善服务性能，可靠性和弹性 查找对任何类型资源的过多或不必要的请求。例如：反复加载的文件，以多个版本加载的 JavaScript，从不使用的 CSS，从不查看（或可以延迟加载）的图像。
优化内存使用 内存可能成为新的瓶颈 (opens new window)，尤其是在移动设备上 加载首页和使用其他网站功能时，请使用 Chrome 任务管理器将您的网站与其他网站进行比较，以了解内存使用情况。
减少 CPU 负载 移动设备的 CPU 有限，尤其是低规格设备 检查是否有大量 JavaScript。使用覆盖工具 (opens new window)查找未使用的 JavaScript 和 CSS 。检查是否有过多的 DOM 大小 (opens new window)以及在首次加载时不必要地运行的脚本。寻找加载有多个版本的 JavaScript 或通过较小的重构可以避免的库。

有多种工具和技术可用于审核网站：

  • 系统工具
  • 内置浏览器工具
  • 浏览器扩展
  • 在线测试应用
  • 仿真工具
  • 分析工具
  • 服务器和业务系统提供的指标
  • 屏幕和录像
  • 手动测试

您将在下面了解与每种审核类型相关的方法。

图片构成迄今为止 最重 (opens new window)和 最请求 (opens new window)对于大多数网页。

随着连接性的恶化，延迟 (opens new window)也会越来越严重，因此随着网络的移动，过多的图像文件请求成为一个日益严重的问题。图像还会消耗功率：更多图像请求，更多无线电使用，更多电量耗尽的电池。即使只是渲染图像也要消耗能量 (opens new window) -这与图像的大小和数量成正比。

同样对于内存：像素尺寸的微小增加会导致内存使用量的大幅增加。在移动设备上（尤其是在低规格设备上）使用图像时， 内存可能成为新的瓶颈 (opens new window)。膨胀的图像对于使用上限数据计划的用户也有问题。

删除多余的图像 (opens new window)！如果无法摆脱它们，请进行优化：尽可能提高压缩率，减小像素尺寸，并使用能为您提供最小文件大小的格式。优化诸如横幅和背景之类的“英雄形象”是一次轻松的一次性胜利。

# 记录资源请求：数量，大小，类型和时间

审核站点时，一个不错的起点是使用浏览器的网络工具检查页面。如果您不确定如何执行此操作，请浏览 Chrome DevTools 网络面板 入门指南 (opens new window)。类似的工具可用于 Firefox (opens new window)， Safari (opens new window)， Internet Explorer (opens new window)和 Edge (opens new window)。

进行更改之前，请记住记录结果。对于网络请求，就像截图一样简单-您还可以 将配置文件数据另存 (opens new window)为 JSON 文件。以下是有关如何保存和共享测试结果 (opens new window)的更多信息。

在开始审核网络使用情况之前，请确保 禁用浏览器缓存， (opens new window) 以确保获得有关首次加载性能的准确统计信息。如果您已经通过服务工作者进行缓存，请 清除“缓存 API 存储” (opens new window)。您可能要使用隐身（私人）窗口，这样就不必担心禁用浏览器缓存或删除以前缓存的条目。

以下是一些应使用浏览器工具检查的核心功能和指标：

  • 负载性能：Lighthouse (opens new window) 提供负载指标的摘要。阿迪·奥斯曼（Addy Osmani）撰写了 有关页面加载关键用户时刻 (opens new window)的精彩摘要。
  • 用于加载和解析资源以及内存使用情况的时间轴事件 (opens new window)。如果您想更深入一点，请运行内存和 JavaScript 分析 (opens new window)。
  • 页面总重量和文件数。
  • JavaScript 文件的数量和重量。
  • 任何特别大的单个 JavaScript 文件（例如，超过 100KB）。
  • 未使用的 JavaScript。您可以使用 Chrome Coverage 工具 (opens new window)进行检查 。
  • 图像文件的总数和权重。
  • 任何特别大的单个图像文件。
  • 图像格式：是否有 可能是 JPEG 或 SVG 的 PNG (opens new window)？WebP 是否与后备一起使用？
  • 是否使用响应式图像技术（例如 srcset (opens new window)）。
  • HTML 文件大小。
  • CSS 文件的总数和重量。
  • 未使用的 CSS。（在 Chrome 浏览器中，使用 覆盖面板 (opens new window)。）
  • 检查其他资产（例如 Web 字体（包括图标字体））的使用是否有问题。
  • 检查 DevTools 时间轴是否有任何阻碍页面加载的内容。

如果您使用的是快速 WiFi 或快速的蜂窝连接，请使用低带宽和高延迟仿真进行测试 (opens new window)。记住要在移动设备和台式机上进行测试-有些站点使用 UA 嗅探为不同的设备提供不同的资产和布局。您可能需要使用远程调试 (opens new window)在实际硬件上进行测试 ，而不仅仅是使用设备仿真。

您通常可以使用浏览器工具来发现问题，只需检查网络响应并按大小排序即可。

例如：此处的 349KB PNG 看起来可能是一个问题：

果然，结果是图像为 1600px 宽，而元素的最大显示宽度仅为 400px。解压缩后，图像需要超过 4MB 的内存，这在手机上是很大的。

# 检查内存和 CPU 负载

进行更改之前，请记录内存和 CPU 使用率。

在 Chrome 中，您可以从“窗口”菜单访问任务管理器。这是检查网页要求的简单方法。

Chrome 的任务管理器-注意内存和 CPU 消耗！

# 测试第一个和后续负载性能

Lighthouse (opens new window)， WebPagetest (opens new window)和 Pagespeed Insights (opens new window)可用于分析速度，数据成本和资源使用情况。WebPagetest 还将检查静态内容缓存，到第一个字节的时间以及您的站点是否有效利用了 CDN。

启用静态内容缓存非常简单，因此浏览器可以在第一次请求资产时通过配置服务器以包括适当的标头来缓存资产。

如果浏览器可以缓存资源，则在以后的访问中无需从网络检索资源。即使对于不支持通过服务工作者进行缓存的浏览器，这也可以提高加载速度，降低数据成本并减少网络和服务器负载。 即使您使用的是 Cache API (opens new window)，启用浏览器缓存也很重要。

要了解更多信息，请查看 PageSpeed 工具 (opens new window)和Web 基础知识 (opens new window)上的资源 （特别是“无效和更新缓存的响应”部分）。

# 测试核心渐进式 Web App 要求

Lighthouse (opens new window)可帮助您测试安全性，功能性，可访问性，性能和搜索引擎性能。特别是，Lighthouse 会检查您的站点是否成功实现了 PWA 功能，例如服务工作者和 Web App 清单。

Lighthouse 还测试您的站点是否可以提供可接受的脱机体验。

您可以以 JSON 格式下载 Lighthouse 报告，或者，如果您使用 Lighthouse Chrome Extension (opens new window)，则以 GitHub Gist 的形式共享报告：单击共享按钮，选择在 Viewer 中打开，然后在新窗口中再次单击共享按钮和另存为要点。

将报告从 Lighthouse Chrome 扩展程序导出到要点-单击“共享”按钮

# 使用分析，事件跟踪和业务指标来跟踪实际绩效

如果可以，请在实施更改之前记录分析数据：跳出率，页面停留时间，退出页面：与业务需求相关的任何内容。

如果可能，记录可能受到影响的业务和技术指标，以便您可以在进行更改后比较结果。例如：一个电子商务站点可能会跟踪每分钟的订单或记录统计数据以进行压力和耐力测试。如果减少页面重量和资源请求，则后端存储成本，CPU 需求，服务成本和弹性可能会提高。

如果没有实施分析，现在是时候了！业务指标和分析是决定您的网站是否正常运行的最终决定因素。如果合适，请结合 事件跟踪 (opens new window) 来执行用户操作，例如按钮单击和视频播放。您可能还需要实施 目标流分析 (opens new window)：用户导航到“转化”所依据的路径。

您可以随时关注 Google Analytics（分析） 网站的速度， (opens new window)以检查效果指标与业务指标之间的关系。例如：“首页加载速度有多快？” 与“通过首页输入是否产生销售？”相比

Google Analytics（分析）使用Navigation Timing API 中的 (opens new window)数据 。

您可能想要使用JavaScript 性能 API 之一 (opens new window) 或您自己的指标来记录数据 ，例如：

const subscribeBtn = document.querySelector('#subscribe');

subscribeBtn.addEventListener('click', (event) => {
 // Event listener logic goes here...

 const lag = performance.now() - event.timeStamp;
 if (lag > 100) {
  ga('send', 'event', {
   eventCategory: 'Performance Metric'
   eventAction: 'input-latency',
   eventLabel: '#subscribe:click',
   eventValue: Math.round(lag),
   nonInteraction: true,
  });
 }
});

您还可以使用 ReportingObserver 来检查浏览器过时和干预警告。这是从实际用户那里获取真实，实时测量结果 (opens new window)的众多 API (opens new window)之一。

# 真实体验：屏幕和视频录制

对移动设备和台式机上的页面加载进行录像。如果添加了计时器显示，则在高帧频下效果更好。

您可能还想保存截屏视频。有许多适用于 Android，iOS 和桌面平台的截屏录制应用程序（和 脚本也可以这样做 (opens new window)）。

视频记录页面加载的工作方式 (opens new window)与 WebPagetest 中的 幻灯片视图 (opens new window)或 Chrome DevTools 中的Capture 屏幕截图 (opens new window)非常相似。您将获得页面组件加载速度的真实记录：快和慢。保存视频记录和屏幕录像，以与以后的改进进行比较。

前后比较可以很好地证明改进！
