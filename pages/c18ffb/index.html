# 介绍

现代的 Web 应用程序经常使用捆绑工具来创建文件（脚本，样式表等）的生产“捆绑包”，该捆绑包 经过优化 (opens new window)， 最小化 (opens new window) ，用户可以在较短的时间内下载。在使用 webpack 进行 Web 性能优化中，我们将逐步介绍如何使用webpack (opens new window)有效地优化站点资源。这可以帮助用户更快地加载您的网站并与您的网站进行交互。

webpack 是当今使用的最受欢迎的捆绑工具之一。利用其功能优化现代代码， 将 (opens new window) 脚本分为关键和非关键部分，并剥离未使用的代码（仅举几例优化），可以确保您的应用具有最小的网络和处理成本。

灵感来自 捆绑伙伴中 (opens new window)的代码拆分， (opens new window)作者：Susie Lu

**注意：**我们创建了一个训练应用程序，以实现本文所述的优化功能。尝试从中获得最大收益以练习提示： webpack-training-project (opens new window)

让我们开始研究如何优化现代应用程序中最昂贵的资源之一-JavaScript。
